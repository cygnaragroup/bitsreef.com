02 · How it works

From git push to a public URL.

Four steps. No proxy config. No certificate dance. No "works on the build server" surprises.

step 01 ~ 10s

Connect a repo

GitHub or GitLab OAuth. Pick branch and root, set auto-deploy on push.

acme/checkout-api
branch · main
auto-deploy · on
step 02 ~ 1–2 min

Kaniko builds

Image built in a non-privileged container, layer-cached, pushed to the internal registry.

✓ step 1/6 · FROM node:20-alpine
✓ step 2/6 · COPY . /app
✓ step 3/6 · RUN npm ci
● step 4/6 · RUN npm run build
cached · 412 MB · pushed in 11s
step 03 ~ 30s

Traefik routes

New containers come up. Health-checked, then Traefik shifts traffic. Let's Encrypt cert issued in the same step.

drain30s
tlsletsencrypt · ECDSA-256
old · v0.4.1 (drained) → new · v0.4.2 (live)
step 04 ∞

Live, autoscaled

Metrics streaming. Autoscaler watching. Rollback waiting in case the next push lies to you.

03 · Features

One platform. The whole stack.

Containers, functions, cron, storage, networking, autoscale, alerts, billing. Built together — not stitched.

Deploy three sources, one button

Git, image, template — same flow.

Connect a repo, point at any registry image, or pick a template (Postgres, Redis, Mongo…). The next screen is the same dashboard.

Git

Image
docker hub · ghcr · ecr · gcr · private registries
Template
postgres · redis · mongo · mysql · rabbitmq · minio
Resilience zero-downtime + auto-rollback

If it fails, it goes back.

Health-checked rollouts. Drain old containers. Failed deploy → automatic rollback to the last green version. One-click to any previous deploy.

Networking

Custom domains, auto-SSL, private DNS.

Every service gets a subdomain. Add your own — DNS verified, cert issued, renewed forever.

Autoscale

Up. Down. To zero.

Horizontal autoscaler evaluates every 30 seconds. Quiet services scale-to-zero and wake on the first request.

2 ↗ 8 replicas scale-to-zero · 15m idle
Observability

Live logs, parsed.

WebSocket tail across every replica. JSON auto-parsed and severity-coloured. Search the whole org with one query.

14:22:03 WARN node-04 cpu 87%

14:22:05 retry in 4s · 2/5

Functions

One-file workloads in Bun, Deno, Python, or Node. HTTP-triggered or cron. X-Function-Key for external systems.

S3 buckets

MinIO inside, S3 API outside. Public buckets, per-bucket keys, in-browser file explorer.

Cron jobs

Run commands inside your running containers on a schedule. Output, exit code and duration logged per run.

Teams + audit

Viewer / Member / Admin / Owner. Every mutation written to an immutable audit log with IP and timestamp.
